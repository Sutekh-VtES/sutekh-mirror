Introduction

If you are new to Sutekh, try reading The Sutekh Tutorial.

Important Concepts

Sutekh is concerned with two different types of objects: Cards and Card Sets.

Cards are the individual cards as described by the official white wolf card list. Each card describes all the relevant properties of the card, such as the clan and discipline requirements, the cost, and the latest text for the card.

Card Sets are collections of cards and describe the card and the number of copies of that card in the collection. An important concept is that card sets can be related in a parent-child hierarchy. A typical setup is to have a single card set for the main card collection (usually the My Collection card set), and other card sets, which represent the constructed decks, as children of this card set, but many other arrangements are possible.

Main Window

The main sutekh window can be split into a number of different panes.

The following panes are supported:

  • The White Wolf card list
  • The Card Set List
  • Card Text
  • Card Image
  • Card Set panes

Each pane may have an associated menu for actions specific to that pane. The main window has a menu for global actions.

Panes can be easily rearranged. Dragging a pane onto another pane will swap the two panes.

Pane Actions

This menu allows you to manage the panes in Sutekh. The Pane Actions menu has the following options:

  • Equalize pane sizes: This rearranges the panes so that each pane has the same width, and panes which are split vertically all have the same height.
  • Split current pane horizontally: This splits the currently active pane into the currently active pane and a new pane to the immediate right of the active pane. The two panes of the same height, and half the width of the pane being split.
  • Split current pane vertically: This splits the currently active pane into the currently active pane and a new pane below the active pane. The two panes of the same width, and half the height of the pane being split. It is not possible to split panes vertically multiple times.
  • Add New Pane: This adds a new pane of the chosen type to the far right of the main window.
  • Replace Pane: This opens a submenu, where the currently active pane can be replaced with any of the available options.
  • Close Pane: This closes the currently active pane

Keyboard Shortcuts

The following shortcuts are useful for most panes:

One can cycle between different panes using the <Tab> key.

The currently active pane can be closed using the <Ctrl>-W shortcut.

For panes which support filtering, the filter editing dialog can be activated using <Ctrl>-S (for specify filter), and the current filter can be toggled on and off by using <Ctrl>-T.

The <Ctrl>-+ and <Ctrl>— shortcuts allow the entire tree displayed in a pane to either be fully expanded or completely collapsed.

Keyboard Shortcuts

For card sets, the following additional keyboard shortcuts exist.

The editable state of the card set can be toggled using <Ctrl>-E.

The current selection can be copied to the clipboard using <Ctrl>-C

If the card set is editable, the current selection can be deleted using <Del> or pasted into the card set using <Ctrl>-V

For an editable card set, the card number for the entire selection can be set in the range 1 to 9 directly by entering the number directly, or all the number for all cards in the selection can either be incremented or decremented by 1 using either + or - respectively. If the card set is not editable, these keys have no effect.

Backup

This allows you to either save or restore a full backup of the current database. The backup will be saved as a zip file and is independent of the database back end Sutekh uses. Consequently, the backup can also be used to transfer the complete state between different databases.

There are two options offered by the plugin:

  • Save a Full Backup
  • Restore a Full Backup

Save a Full Backup will save the current state (card sets and their contents) to the specified zip file. Restore a Full Backup will import all the card sets from the backup. Note that restoring a backup will replace anything currently in the database with the contents of the backup.

Backups can be saved at any time. Various actions, such as updating the WW card list, will also allow a backup to be saved.

Change Grouping

By default, Sutekh groups the cards into the separate card types. The grouping used can be changed via this menu option.

The following groupings are available:

  • Clans and Creeds: Group cards by the either clan or the creed associated with the card. This treats the Imbued Creeds as if they were Vampire Clans.
  • Disciplines and Virtues: Group cards by the either the disciplines or the virtues on the card. This treats the Imbued Virtues as if they were Vampire Disciplines.
  • Crypt or Library: Split the card list into crypt and library cards
  • Card Type: Group by card type. This is the default grouping. This grouping treats multi-type cards, such as “Action Modifier/Combat” cards as belonging to both the “Action Modifier” and the “Combat” groups.
  • Expansion: Group by the expansions the cards have been printed in
  • Rarity: Group the cards by the published rarity.
  • Sect: Group cards by the Sect associated with the card.
  • Title: Group cards by the political title on the card.
  • Multi Card Type: Like the Card Type grouping, except this treats multi-type cards as separate types. Thus “Action Modifier/Combat” will appear as a separate group, and the cards will not be listed in either the “Action Modifier” or “Combat” groups.

Select Extra Columns

By default, sutekh only shows the card name for the White Wolf Card List, and the number and card name for the card set lists. Additional columns to display can be selected using this plugin. The extra columns are:

  • Clans and Creeds: Also display the clans or creeds associated with the card.
  • Group: Show the group number for the card.
  • Disciplines and Virtues: Show the Disciplines and Virtues associated with the card.
  • Card Type: Also show the card type.
  • Expansions: Show the expansions this card has been printed in.
  • Capacity or Life(Imbued): Show the capacity associated with this card, or the life for the Imbued crypt cards. Ally & retainer life is not shown.
  • Sect: Show the Sect associated with the card.
  • Title: Show the political title possessed by the vampire card.
  • Card Text: Show the text printed on the card.

The display can be sorted by according to these columns, by clicking on the column header.

If the icons have been downloaded from the white wolf site, then for the appropriate columns, the display can be toggled between the Show Icons and Names, Show Text only, Show Icons only using the radio buttons. This option will affect all the selected columns that can use icons.

Simulate Opening Hand

This plugin treats the card set as a completed deck, and displays the expected distribution of cards in the opening hand and crypt. It is intended to give some idea of how the deck will work in practice. In addition, the plugin can generate example opening hands and crypts via the _Draw sample hand_ button.

Card draw probabilities

This plugin displays the probability of drawing the selected cards from either the library or the crypt. Unlike the simulate opening hand plugin, this plugin displays results for longer sequences of drawing cards.

The plugin has the following settings which can be used to change the results.

  • columns in table: This sets the number of draws the columns with probabilities shown in the table and can range from 1 to 8. The first column is always the probably for the opening hand or crypt draw. Subsequent columns are the total probability after also drawing the number of extra cards shown in the column header.
  • step between columns: This sets the number of cards drawn between columns and ranges from 1 to 10 (defaulting to 1). Each subsequent column after the first of the table shows the probabilities after drawing this number of additional cards.
  • cards of interest: This sets the number of rows in the table, and can range from 1 to the total number of cards selected.

The selected changes are only applied when the recalculate table button is pressed.
