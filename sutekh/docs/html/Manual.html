Introduction

If you are new to Sutekh, try reading The Sutekh Tutorial.

Important Concepts

Sutekh is concerned with two different types of objects: Cards and Card Sets.

Cards are the individual cards as described by the official White Wolf card list. Each card describes all the relevant properties of the card, such as the clan and discipline requirements, the cost, and the latest text for the card.

For each card, the expansion that specific card is from can be set. This allows one to distinguish between multiple copies of the same card from different expansions. If the expansion is not set, this is noted as Unknown Expansion in the card set view.

Card Sets are collections of cards and describe the card and the number of copies of that card in the collection. An important concept is that card sets can be related in a parent-child hierarchy. A typical setup is to have a single card set for the main card collection (usually the My Collection card set), and other card sets, which represent the constructed decks, as children of this card set, but many other arrangements are possible.

Main Window

The main sutekh window can be split into a number of different panes.

The following panes are supported:

  • The White Wolf card list
  • The Card Set List
  • Card Text
  • Card Image
  • Card Set panes

Each pane may have an associated menu for actions specific to that pane. The main window has a menu for global actions.

Panes can be easily rearranged. Dragging a pane onto another pane will swap the two panes.

File

The file menu has the following options.

  • Import Card Set: Import a card set from a existing file. The following file types are supported:
    • Load Card Set from File: Loads a card set saved in Sutekh’s XML file format.
    • Import CSV File: Imports a card set from a comma-separated value file, as produced by a spreadsheet. This allows the columns for card set, number and expansion to be selected.
    • Import ARDB or ELDB Card Set: Imports a file saved by either ARDB and FELDB. Note that the correct type of file must be specified as well as the file name.
    • Import Card Sets from zip file: Allows one to select card sets from a zip file to import.
  • Import new White Wolf Card List and rulings: Update the database with a new cardlist and/or rulings file. The files can either be local, or they can be downloaded from the White Wolf website directly. There may be issues when updating the card list after a new expansion has been published, discussed in the section “Refreshing the White Wolf Card List”#refresh
  • Download VTES icons: Attempt to download the icons from the White Wolf website.
  • Preferences: Manages various preferences.
    • Save Current Pane Set: Saves the current configuration of the panes.
    • Save Pane Set on Exit: Saves the pane configuration when Sutekh exits.
    • Save Exact Pane Positions: Saves the exact pane positions on exit.
    • Save Window Size: Save the window size on exit.
  • Restore saved configuration: Restores the panes to the saved configuration.
  • Quit: Exit Sutekh

When importing card sets, if Sutekh encounters an unknown card, the Card Lookup Window is used to identify them.

Pane Actions

This menu allows one to manage the panes in Sutekh. The Pane Actions menu has the following options:

  • Equalize pane sizes: This rearranges the panes so that each pane has the same width, and panes which are split vertically all have the same height.
  • Split current pane horizontally: This splits the currently active pane into the currently active pane and a new pane to the immediate right of the active pane. The two panes are the same height as, and half the width of the pane being split.
  • Split current pane vertically: This splits the currently active pane into the currently active pane and a new pane below the active pane. The two panes are the same width as, and half the height of the pane being split. It is not possible to split panes vertically multiple times.
  • Add New Pane: This adds a new pane of the chosen type to the far right of the main window.
  • Replace Pane: This opens a submenu, where the currently active pane can be replaced with any of the available options.
  • Close Pane: This closes the currently active pane

Keyboard Shortcuts

The following shortcuts are useful for most panes:

One can cycle between different panes using the <Tab> key.

The currently active pane can be closed using the <Ctrl>-W shortcut.

For panes which support filtering, the filter editing dialog can be activated using <Ctrl>-S (for specify filter), and the current filter can be toggled on and off by using <Ctrl>-T.

The <Ctrl>-+ and <Ctrl>— shortcuts allow the entire tree displayed in a pane to either be fully expanded or completely collapsed.

Card Lookup Window

There are various reason why Sutekh my fail to correctly identify a card when importing a card set. Names vary (“Spirit’s Touch”, “The Spirit’s Touch”), special characters can be missing, and so forth. When this happens, Sutekh prompts for the correct card.

The card lookup window displays two lists – the list of all white wolf cards and the list of unknown cards. Below the unknown card list are options to specify a filter and to apply or unapply the filter.

The unknown card list has 2 columns, the name of the unknown card and the current selection to replace the card, and 3 buttons. The first button uses the currently selected card from the full White Wolf card list to replace the card, the next marks the card as to be ignored, and the last applies a filter based on the unknown card name to present a list of reasonable options.

Once the replacement cards have been selected, the OK button will create the correct card set.

Choosing cancel in the card lookup dialog will abort the entire import, and no card set will be created.

If Sutekh cannot identify an expansion in the file, a dialog will prompt for the correct replacement. If an impossible card and expansion combination is found in the file, the same card lookup dialog will be used, except that the expansion as well as the card can be set.

Refreshing the White Wolf Card List

This allows Sutekh to update the card list to account for new cards, such as promos and such.

Since Sutekh parses the card list published on the white wolf site for information about the cards, when new expansions are published, Sutekh may require an update to correctly interpret any changes to the cardlist, such as new clans or card types. If the expansion does not introduce any significant changes, Sutekh is able to correctly import the new card list, but may not interpret the new expansion name correctly. If this happens, Sutekh should work fine, but, when updating to a new version of Sutekh, one may need to specify the correct mapping for the expansion when updating the database.

Card Text

The Card Text Pane displays the text on the card, and any rulings that apply to the card.

This will also list other relevant information about the card, such as the card cost, the card type, the disciplines or virtues associated with the card, the clans and creeds associated with the card, vampire titles, sects and groups, capacity and the expansions in which the card was printed.

Card Images

If the Card Image Plugin has been configured, this pane will display the appropriate card image for each pane. By default, the image is resized to fit the pane. By right clicking on the image, this can be changed. The following options are available:

  • Fit images to the pane: This is the default
  • Show images at fixed size: All the images are shown as 225×300 in size.
  • Show images at original size: All the image are shown at the original size of the scan.

If there are multiple images for the given card form the different expansions, Sutekh will attempt to display the correct expansion for the selected card. It’s possible to cycle through the available expansions by either left clicking on the image, or using the Show next expansion image and _Show previous expansion image_ options when right clicking on the frame.

Import CSV file

This allows importing card sets stored as a comma-separated file, commonly produced by spreadsheets. The file must contain a column with card names, and a column with the card counts, and can optionally contain a column with the expansions. When importing the file, the mapping between columns and the contents needs to be specified.

If Sutekh cannot correctly interpret a name or expansion, the Card Lookup Window will be used to resolve the situation.

Import ARDB or ELDB file

This allows one to import files produced by ARDB or FELDB. The file format is not automatically recognised, so it needs to be manually specified.

For some types of file, such as ARDB inventories, the card set name for the imported file will need to be specified manually.

If the name of the card set clashes with an existing name, one will be asked to rename the imported card set or cancel the import.

If no cards are found in the card set, the card set will not be created.

Import card sets from zip file

This plugin allows one to select some of the Sutekh Card Sets included in a zip file (such as those produced by the backup plugin) and import them into the card set. Unlike the backup plugin, this does not replace any of the existing card sets.

After selecting the zip file, one will be asked to select the card sets to import from the list of card sets in the zip file. If any of the card sets share the same name as an existing card set, one will be asked to either rename the card set, or skip importing that card set.

For card sets with parents, if the appropriate parent card set can’t be found, the card set will be imported with no parent set.

    • Show difference between Parent Count and card sets in use: Show difference between parent count and all total amongst all the card sets marked in use. If this number is negative, it is shown in red.

Filters

Filtering is one of the central activities in Sutekh. Complex queries can be constructed by combining the various basic filters.

The Filter dialog will display the last used filter for the given pane. This filter can then be modified or the filter can be cleared using the Clear Filter button.

Complex filters can be saved for later reuse using the Save button, and recalled using the Load button. When modifying a filter, it can be reverted to the saved version using the Revert Filter button.

Card Filters

These filters apply to either the white wolf card list or a card set. The following basic filters are supported:

  • Card Type: Filters on the type of card. This takes a list of card types.
  • Cost Type: Filters on the type of cost (blood, pool, and so forth). This takes a list of cost types. This does not consider the cost, just the cost type.
  • Clan: filter on the clan(s) associated with the card. This takes a list of clans and the filter matches if the clan matches any clan in the list.
  • Discipline: Filter on the discipline associated with the card. This takes a list of disciplines, and the filter matches if any of the disciplines in the list match. This filter ignores the level requirement for the discipline.
  • Group: Filter on the group listed on the card. This takes a list of groups, and matches the group listed on the card to this list.
  • Capacity: Filter on the capacity listed on the card. This takes a list of capacities, and matches the capacity listed on the card to this list.
  • Cost: Filter on the cost of the card. This filter takes a list of costs to match, and matches the numerical card cost to this list. Note that this does not consider the cost type. To filter only on costs of a specific type, this needs to be combined with the Cost Type filter.
  • Life (Crypt): Filter on the life cost of the crypt card. This takes a list of values, and matches the life listed on the card to this list. Note that this only matches the life associated with crypt cards (currently the Imbued), and doesn’t match the life for other allies or retainers.
  • Creed: Filter on the creed associated with the card. Like the Clan filter, this takes a list of creeds to match.
  • Virtue: Filter on the virtues associated with the card. Like the Discipline filter, this takes a list of virtues to match.
  • Card Text: Filter on the text of the card. This filter takes a string to match against.
  • Card Name: Filter on the name of the card. This filter takes a string.
  • Sect: Matches against the sect of the vampire. This filter takes a list of sects to match against.
  • Title: Filter on the title of the vampire. This filter takes a list of titles and matches the card title to this list.
  • Expansion_with_Rarity: Filter on the expansion and rarity of the card. This filter takes a list of expansion with rarities, and matches a card if it was printed with the given expansion and rarity combinations. Note that this filter matches the list of expansions a card has been printed in, and ignores the expansion information set for a card. If one wants to filter on the card’s expansion information, use the Physical Expansion filter.
  • Discipline with Level: Filter on the discipline associated with the card. This takes a list of disciplines of the different levels, and the filter matches if the card matches the disciplines in the list. This filter treats the different levels of the filter as different. This filter is most useful for matching vampires. Note that non-vampire cards are matched only by the superior discipline. inferior.
  • Physical Expansion: This filter matches on the expansion information set for a card. This filter takes a list of expansions, and matches the expansion set for the card to this list. This is different from the Expansion_with_Rarity filter since it ignores rarities, and matches the expansion set for the card, not the list of expansions the card was printed in.
  • Card Sets: The filter matches cards that belong to the given list of other card sets.
  • In Card Sets In Use: This matches the cards against the child card sets marked as in use. A card matches if it belongs to a card set in use. This filter has no parameters.
  • Card Count: This matches the card against a count in either the same card set or a different card set. It takes a list of card counts, and a single card set to match against, and matches those cards that satisfy these conditions.
  • Card Function: This filter matches the card against a list of common card functions.
  • Sub-Filter: Nest an additional filter box inside the current box. See Combining Basic Filters

These filters are also available in the Card Lookup Window when importing card sets, to help find unknown cards.

Currently, the only way to filter on properties of a merged vampire, such as different discipline levels, is to filter on the card text.

Card Set Filters

These filters apply to the card set list. The following basic filters are supported.

  • Card Set Name: This matches the name of the card set. Like the Card Name or Card Text filters, it is treated as a partial string to search for. % and _ can be used as wildcard characters.
  • Card Set Description: This matches the description of the card set. Like the Card Name or Card Text filters, it is treated as a partial string to search for.
  • Card Set Author: This matches the description of the card set. Like the Card Name or Card Text filters, it is treated as a partial string to search for.
  • Card Set Annotations: This matches the annotations of the card set. Like the Card Name or Card Text filters, it is treated as a partial string to search for.
  • Parent Card Set: This takes as input a list of card sets, and matches those card sets whose parent card set is in the list.
  • Card Set Marked as in Use: This matches all card sets which are marked as being In Use.
  • Sub-Filter: Nest an additional filter box inside the current box. See Combining Basic Filters

Since the card sets can be part of a hierarchy of parents and children, if a child card set matches the filter, it will be displayed along with it’s parents. If a parent card set matches the filter, only those children that also match the filter will be displayed.

Combining Basic Filters

Sutekh combines filters using filter boxes to group the basic filters together.

There are 4 types of filter box, namely:

  • All of: This filter matches only if every filter in the box matches.
  • Any of: This filter matches if at least one of the filters in the box matches.
  • Not all of: This filter matches if at least one of the filters in the box doesn’t match.
  • Not any of: This filter matches only if every filter in the box doesn’t match.

Each filter boxes can also contain sub-filters, which are themselves filter boxes. Thus complex queries can be constructed by nesting multiple filter boxes.

String filters

Several filters use a user specified string to match on. These filter matches if this string is contained in the property filtered on, so a Card Name filter, if the given string is a substring of the card name.

String filters are also case insensitive (so the will match both *Weigh the Heart* and Yuri, The Talon, when filtering on the card name, for example).

In Addition, filters which take a string as input can use two different wildcard characters.

_ will match any single letter. Thus a filter using a_c will match both abc and aec, but not match abdc or abd. _ will never match the empty string, so Th_e will not match The, but will match Thee

% will match any selection of characters. So a%c will match any of abc, aec, or abdegfthc, but not match abdefg. % will also match the empty string, so Th%e will match both The and Three.

Wildcards can be used multiple times. So a_b%e will match any of acbde, aebe or aebfe.

List filters

Several filters accept a list of possible options to match against.

The filter is considered to match if any of the entries in this list are matched. Thus a Clan filter, specifying both Tremere and Toreador, will be matched by cards requiring/of either of the clans.

Plugin List

Backup

This allows one to either save or restore a full backup of the current database. The backup will be saved as a zip file and is independent of the database back-end Sutekh uses. Consequently, the backup can also be used to transfer the complete state between different databases.

There are two options offered by the plugin:

  • Save a Full Backup
  • Restore a Full Backup

Save a Full Backup will save the current state (card sets and their contents) to the specified zip file. Restore a Full Backup will import all the card sets from the backup. Note that restoring a backup will replace anything currently in the database with the contents of the backup.

Backups can be saved at any time. Various actions, such as updating the White Wolf card list, will also allow a backup to be saved.

Card Images

This plugin allows configuring the card images pane.

The image can be configured in three ways.

  • Download and install cardimages from feldb.extra.hu: This attempts to download the images form the FELDB website, and makes them available to Sutekh. The zip file is quite large, so this option may take some time.
  • Install images from a local zip file: If a zip file containing the images is available, Sutekh can install directly from this zipfile.
  • Choose a directory containing the images: If the images have already been installed for some other reason (such as an existing ARDB or FELDB installation), then this option can be used to configure Sutekh to reuse that installation.

Cluster Cards

The clustering plugin attempts to group cards from a card list into sets (clusters) of cards with similar properties. The intention is to provide a way to explore subsets of one’s cards and perhaps to look for groups of cards which might provide starting points for deck construction. Currently running the clustering tool is quite slow, so be patient when waiting for large card lists to be grouped (it will finish eventually).

Upon opening the clustering plugin, three tabs are presented— Select Columns, Settings and Results.

The Select Columns tab is where one selects the properties to consider when grouping the cards. Properties are organised into categories for ease of selection. If unsure on which properties to select, start by selecting the disciplines and clans you’re most interested in.

Under the Settings tab, one can tweak the clustering algorithm parameters. The plugin implements K-means clustering using K-means++ to determine the initial cluster centers followed by Lloyd’s method of iteratively refining the clusters. Three parameters can be set:

  • Number of iterations: Number of Lloyd refinement steps to perform. More steps makes the clustering take longer but fewer steps may result in a less optimal grouping. Ten steps should be sufficient for most cases.
  • Number of clusters: By default the plugin creates one cluster per 80 cards (this being the size of a deck) but the number of clusters may also be set manually.
  • Distance measure: Two distance metrics are currently supported. The Euclidean distance setting clusters using the usual N-dimensional vector space metric. The Sutekh distance metric modifies the Euclidean one to make property values of -1 (used internally by Sutekh to mark costs of X and the ANY crypt group) close to all others and values of zero slightly further away than usual. If in doubt, leave the metric set to the Sutekh distance.

Once the property columns have been selected and the parameters set, use the Execute button to begin the clustering. When clustering is complete, the Results tab will be populated.

The results are presented as a table. Each row corresponds to a cluster of cards. The first column contains checkboxes. Use these to select which clusters to create cards sets from before clicking the Make Cards Sets from Selected Clusters button at the bottom of the results. The second column contains the cluster number, which enables one to determine which cluster the card sets created correspond to. The third column shows how many cards are present in each cluster and the last lists the property values of the cluster center (property values of zero are omitted).

Clan Discipline Stats

This plugin shows an analysis of all the difference clans in the White Wolf card list. For each clan, the number of vampires in each clan, the sum of the vampire capacities and the most common disciplines are shown. The score for the discipline, and various stats about how the disciplines are distributed are also shown.

The tree view can be expanded to show the details for each legal grouping combination.

The score is intended to give some indication of how common disciplines are. Vampires without the discipline score 0 for that discipline, vampires with the inferior discipline score 1 and vampires with the superior discipline score 2.

Change Grouping

By default, Sutekh groups the cards into the separate card types. The grouping used can be changed via this menu option.

The following groupings are available:

  • Clans and Creeds: Group cards by the either clan or the creed associated with the card. This treats the Imbued Creeds as if they were Vampire Clans.
  • Disciplines and Virtues: Group cards by the either the disciplines or the virtues on the card. This treats the Imbued Virtues as if they were Vampire Disciplines.
  • Crypt or Library: Split the card list into crypt and library cards
  • Card Type: Group by card type. This is the default grouping. This grouping treats multi-type cards, such as “Action Modifier/Combat” cards as belonging to both the “Action Modifier” and the “Combat” groups.
  • Expansion: Group by the expansions the cards have been printed in
  • Rarity: Group the cards by the published rarity.
  • Sect: Group cards by the Sect associated with the card.
  • Title: Group cards by the political title on the card.
  • Cost: Group cards by the cost on the card.
  • Multi Card Type: Like the Card Type grouping, except this treats multi-type cards as separate types. Thus “Action Modifier/Combat” will appear as a separate group, and the cards will not be listed in either the “Action Modifier” or “Combat” groups.

Select Extra Columns

By default, sutekh only shows the card name in the White Wolf Card List, and the number and card name for the card set lists. Additional columns to display can be selected using this plugin. The extra columns are:

  • Clans and Creeds: Also display the clans or creeds associated with the card.
  • Group: Show the group number for the card.
  • Disciplines and Virtues: Show the Disciplines and Virtues associated with the card.
  • Card Type: Also show the card type.
  • Expansions: Show the expansions this card has been printed in.
  • Capacity or Life(Imbued): Show the capacity associated with this card, or the life for the Imbued crypt cards. Ally & retainer life is not shown.
  • Sect: Show the Sect associated with the card.
  • Title: Show the political title possessed by the vampire card.
  • Card Text: Show the text printed on the card.
  • Cost: Show the cost of the card.

The display can be sorted according to a column, by clicking on the column header. When cards have equal values, they are sorted by the card name. Because the different cost types aren’t comparable, the cost column groups each distinct cost type together and sorts within the cost types.

If the icons have been downloaded from the white wolf site, then for the appropriate columns, the display can be toggled between the Show Icons and Names, Show Text only and Show Icons only using the radio buttons. This option will affect all the selected columns that can use icons.

Set selected cards to a single expansion

This searches for the common expansions between all the selected cards, and allows the user to choose one of these. The physical expansion for all the selected cards is set to this common choice.

Compare card set with another Card Set

This plugin compares the current card set to a different card set, and displays which cards are common to the card sets, and, for each card set, lists the unique cards.

By default, the plugin ignores the expansions of cards, and considers all cards of the same name as the same, but this behaviour can be changed using the Consider Expansions as well checkbox when selecting the card set to compare to.

Analyze Deck

This plugin examines a card set and displays various statistics about it that are intended to give some indication of how the card set will operate as a deck.

Several tabs are displayed, where each tab summarises different aspects of the card set. The first tab summarises some basic information about the entire set. The second tab implements Legbiter’s Happy Family analysis for the card set. Subsequent panes consider specific card types.

The happy family implementation allows one to either select the number of disciplines to use, or to select the disciplines to consider manually.

Test Card Set Independence

This tests the current card set against other card sets with the same parent, and considers whether there are enough cards in the parent card set to simultaneously construct all the selected sets. Multiple decks can be selected from the list of card sets.

The current card set can be tested against all the sets with the same parent marked as in use using the Test against all card sets marked as in use check box when selecting decks to compare against. This check box overrides any selected entries from the list.

The results list all the cards missing from the parent card set, and, for each set which includes the missing cards, the cards for that set are listed.

Simulate Opening Hand

This plugin treats the card set as a completed deck, and displays the expected distribution of cards in the opening hand and crypt. It is intended to give some idea of how the deck will work in practice. In addition, the plugin can generate example opening hands and crypts via the _Draw sample hand_ button.

Card draw probabilities

This plugin displays the probability of drawing the selected cards from either the library or the crypt. Unlike the simulate opening hand plugin, this plugin displays results for longer sequences of drawing cards.

The plugin has the following settings which can be used to change the results.

  • columns in table: This sets the number of draws shown in the table and can range from 1 to 8. The first column is always the probably for the opening hand or crypt draw. Subsequent columns are the total probability after also drawing the number of extra cards shown in the column header.
  • step between columns: This sets the number of cards drawn between columns and ranges from 1 to 10 (defaulting to 1). Each subsequent column after the first of the table shows the probabilities after drawing this number of additional cards.
  • cards of interest: This sets the number of rows in the table, and can range from 1 to the total number of cards selected.

The selected changes are only applied when the recalculate table button is pressed.

Rulebook

This plugin allows downloading copies of the rulebook, imbued rules and card rulings from the White Wolf site or installing previously downloaded copies. Once downloaded, the rulebook files may be opened in a web browser using the Rulebook submenu in the Plugins menu on the Main Window. To download the files, or update the existing copies, use the Configure Rulebook Plugin item in the same submenu.
